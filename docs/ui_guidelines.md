# UI/UXガイドライン

## デザイン哲学

### 必要最小限の情報
**不要な文字もアイコンも削除し、必要な情報だけを表示する**

- 文字、アイコン、装飾は最小限
- ユーザーが理解できる最小の情報量
- 無理にアイコンを付ける必要はない

## アイコン使用例

### マンダラチャートタブ
- **タブアイコン**: `Icons.grid_on` （9×9グリッド）
- **大目標**: 中央配置、大きめのフォント
- **中目標**: 中サイズ、周囲8マス
- **小目標**: 小サイズ、外側8ブロック

### タスクタブ
- **タブアイコン**: `Icons.checklist` （チェックリスト）
- **小目標選択**: 
  - 未選択時: `Icons.flag_outlined` + `...`
  - 選択時: `Icons.flag_outlined` + 小目標名
  - 小目標なし: `Icons.grid_on` → `Icons.arrow_forward` （マンダラチャートへ）

### 期間セクション
各期間にアイコンとラベルを表示：

| 期間 | アイコン | ラベル |
|------|---------|--------|
| 1日 | `Icons.today` | 1日 |
| 1週間 | `Icons.view_week` | 1週間 |
| 1月 | `Icons.calendar_month` | 1月 |
| 1年 | `Icons.event_note` | 1年 |

### アクション
- **追加**: `Icons.add` （各セクションに配置）
- **編集**: タップ（タスクカード全体）
- **削除**: 長押し
- **完了**: チェックボックス（円形）
- **期限**: `Icons.calendar_today` + 日付

### ステータス表示
- **未着手**: グレー
- **進行中**: バッジ表示「進行中」
- **完了**: チェックマーク + 打ち消し線

## 視覚的フィードバック

### 無効状態
- 透明度 0.4
- タップ無効
- スナックバーで理由を簡潔に表示

### 空の状態
- 文字での説明は避ける
- アイコンのみで状態を表現
- 例: 上向き矢印 `Icons.arrow_upward` → 上を選択してください

### タスク数バッジ
- タスクがある場合のみ表示
- 数字のみ、コンパクトに

## カラーシステム

### 役割別の色
- **大目標**: 深い青（`centerBackground`, `centerText`）
- **中目標**: 緑系（`middleBackground`, `middleText`）
- **小目標**: ライトグレー（`smallBackground`, `smallText`）
- **エラー**: 赤（`errorPrimary`）
- **未入力**: 薄いグレー（`emptyBackground`, `emptyBorder`）

### 視覚的な階層
1. 色の濃さで重要度を表現
2. 境界線の太さで階層を表現
3. シャドウで深度を表現

## インタラクション

### ジェスチャー
- **タップ**: 主要アクション（開く、選択）
- **長押し**: 編集・削除などの副次的アクション
- **ドラッグ**: （今後実装予定）

### フィードバック
- アニメーション: `DesignTokens.durationNormal` (250ms)
- 色の変化: ホバー・タップ時
- サイズ変化: 最小限に

## アクセシビリティ

### ツールチップ
文字を最小限にしても、必要に応じてツールチップで説明：
- `IconButton` の `tooltip` パラメータ
- 長押しで追加情報

### コントラスト
- テキストと背景のコントラスト比 4.5:1 以上
- アイコンのサイズは最小 24px

## レスポンシブ

### 画面サイズ対応
- 最大幅: `DesignTokens.screenMaxWidth`
- パディング・マージン: `DesignTokens.space*`
- フォントサイズ: 固定（自動スケーリングなし）

## まとめ

### デザインの5原則
1. **アイコン優先**: 文字は最小限
2. **視覚的階層**: 色・サイズ・位置で重要度を表現
3. **直感的操作**: タップ・長押しで完結
4. **即座のフィードバック**: アクションの結果を瞬時に表示
5. **シンプル**: 必要な情報だけを表示
